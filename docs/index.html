Packet Capture Analysis Tool

PCAP Assistant for Triage

PCAT is an offline CLI for first-pass packet capture analysis. It turns a PCAP into summaries, evidence records, investigation leads, reports, and extracted artifacts that can be handed to Wireshark, TShark, or specialist tools.

What PCAT Is

A briefing and handoff layer for packet captures.

PCAT is not trying to replace Wireshark. It is built for the moment before deep analysis: identify what is inside the capture, surface high-value evidence, explain what is uncertain, and produce commands or files that make the next tool easier to use.

Capabilities

Useful first-pass analysis without hiding uncertainty.

PCAT focuses on repeatable triage output: summaries, evidence, timelines, artifacts, and workflow-friendly exports. Candidate findings stay labeled as candidates.

Capture Summary

Protocol, host, port, DNS, HTTP, TCP stream, and UDP conversation views built from TShark-parsed fields.

Evidence Records

Stable IDs, confidence, previews, frame and stream anchors, and handoff filters for teammate review.

CTF Hunt

Flags, credentials, clue strings, decoded fragments, ICMP payload hints, SYN payloads, and protocol leads.

Artifact Manager

Magic-byte detection with confirmed, candidate, and rejected labels plus validation and skip reasons.

Object Export

Safer carving under `artifacts/`, HTTP object export, and TFTP object export through `pcat extract`.

Structured Reports

HTML, Markdown, TXT, JSON, CSV, evidence JSON, and artifact manifests for repeatable case folders.

Workflow

From unknown capture to next action.

The intended workflow is quick, explicit, and friendly to other tools. PCAT gives a map, not a verdict.

1

Check the environment

Use `pcat doctor` to verify TShark and optional helper tools.

2

Summarize the capture

Run `pcat summary`, `streams`, `dns`, or `http` to establish what traffic exists.

3

Generate evidence

Run `pcat analyze` for the full briefing, stories, findings, reports, and evidence records.

4

Drill down or export

Use `search`, `evidence`, `artifacts`, and `extract` to prepare focused handoff material.

Quickstart

Install locally and run against a capture.

PCAT is currently developed as a Python package from the repository root.

terminal 
python3 -m pip install -e .
pcat doctor
pcat summary -i capture.pcap
pcat analyze -i capture.pcap --ctf --extract -o case-output
pcat evidence -i capture.pcap --top 25 --json
pcat extract -i capture.pcap --http --tftp -o case-output

Scope

Implemented Now

  • Offline capture parsing through TShark.
  • Evidence, findings, stories, timelines, and reports.
  • Artifact validation, manifests, HTTP export, and TFTP export.
  • CTF-oriented hunt workflow and JSON output for automation.

Planned Later

  • Deeper protocol workflows for DNS, HTTP, MQTT, ICMP, and stream reconstruction.
  • External tool integrations such as Zeek and Suricata.
  • Expanded CTF clue normalization after the core workflow is stable.
  • Possible MCP or LLM-facing layer after the analysis core is stronger.
